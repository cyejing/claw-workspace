# 每周行情汇报定时任务排查与修复 2026-06-08 14:25

## 目标
排查为什么"每周行情汇报"定时任务没有把汇报内容发送到飞书。

## 问题根因
1. **isolated session agent 把完整汇报写入了文件**（`task-summary_2026-06-08_1200.md`），但最终返回的 summary 只有一句话："汇报完成，已写入任务总结。老板看下有啥需要调整的？"
2. **cron 的 `announce` 投递机制只发送 summary**，不是文件内容
3. 因此 announce 发送了一句无意义的话，用户实际没收到有用信息
4. 手动触发后同样问题，14:25那次 summary 也只有"汇报已发出，artifact已记录"

## 修复措施
1. **更新定时任务 prompt**：添加明确要求"必须把完整汇报内容直接输出在最终回复中"，禁止用"已写入文件"代替
2. **补发汇报**：通过 `openclaw message send` 命令将今天行情汇报直接发送到飞书
